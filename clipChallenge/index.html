with overall as (
        select
                row_number() over () as row_n,
                *
        from (
                select
                        cast('1' as int) as ix,
                        date_trunc('month',created_at) as transaction_date,

                group by
                        transaction_date
                order by
                        transaction_date
        ) as temp
)
select
        a.transaction_date,
        (cast(a.users as decimal) / b.users) * 100 as overall_retention
from overall a
left join (select * from overall where row_n = ix) as b
on a.ix = b.ix;

What a time-series decomposition pretends to achieve is to translate an observed time-oriented signal into an equation such as:

$$ Y_t = T_t + S_t + R_t $$

Where $Y_t$ corresponds to the observed signal; $T_t$, $S_t$ and $R_t$ represent the trend, seasonal and residual components respectively. Residuals are a portion of the observed value that can not be attributed to nor trend or seasons.

Since we are dealing with a summation of components what we want to obtain is an additive time-series decomposition model. In the following lines of code I am going to implement this kind of model using python.

4. A Clip Executive has come to you and asked that you specifically create 5 - 7 metrics that will give them insight into the business and give them an understanding of how the company is progressing, growing and overall productivity.¶

a. What metrics would you provide and why¶

Diversity of enrolled businesses: Economic diversity is always considered a good thing. It can be interpreted as a measure of resilience; if in a given city every entrepreneur decides to produce flowers and on a given day all bees become extinct all businesses will be at risk. This same argument can be applied to companies that provide financial services, if costumes come from different sectors there are more reasons to expect a guaranteed income as time passes.

One way this key performance indicator could be calculated is by labeling each costumer's activity according to INEGI's DENUE database for the Mexican context to then compute the entropy of the observations; entropy can be interpreted as categorical standard deviation. When a set is homogeneous entropy is low whereas a high entropy corresponds to diversity in a collection of items.

Cost of costumers acquired: There are two relevant aspects in this KPI. The first reveals mere growth, and the second exhibits the effectiveness of the means by which the company is attempting to acquire additional customers. If, for instance, resources are allocated to marketing and publicity only, these costs should eventually decline as potential users are enticed and end up becoming customers. If, on the other hand, no additional costumers enroll this metric would remain rather constant or unchanged.

App openings & average session time per day: This metric can reveal the extent to which users rely on the application for reasons other than making a sale. In other words, users, and especially users from small businesses, could heavily rely on the app for consulting their inventories, getting to know what days and times certain products are sold more often, or even have a general understanding of how their business is faring through data visualization and business intelligence tools. If users value these features, there could indeed be opportunities in terms of new product developments and subscription schemes.

A/B testing outcomes: This metric is more oriented towards evaluating different plans and treatments among different users. An example of this could be setting an experiment in which the user interface of the application is different for a treatment group in order to test whether a new subscription scheme where business intelligence reports are generated on demand for merchants is more likely to occur.

Given that this feature would only be shown to some people that share some characteristics another control group, with these same characteristics, should not be exposed to the newer user interface. Hence, this experiment would aid the decision process of changing certain aspects of the application as long as the test reveals that the treatment has a statistically significant outcome in terms of additional subscriptions to an existing product. Therefore, the metric is the significance of the test that supports that a particular treatment could have an effect on a population (like all customers or members of a given cohort).

However, this kind of metric requires an appropriate experimental design where sample sizes and experiment duration are carefully determined beforehand.

Man-hours per completed task: In project management endeavors a relevant piece of information is how fast or how effectively goals are met. Comparing similar tasks assigned to different groups might reveal bottlenecks that are potentially easy to spot and handle if progress and allocated resources are monitored periodically.

i. why is distinct so important? What would happen if it wasn’t here?¶

If distinct was not specified in that query every single transaction would be returned. However, the transaction date would be displayed in a truncated format where only the year and the month are displayed and this date would always refer to the first date of each merchant's transaction. If, on the other hand, distinct is in place only the first transaction per each merchant will be returned.

It could be cohort_month or first_transaction_date.

i. The calendar table is a simple table that has a row for every day in the year since 1900 - 2050¶

There is an aggregation function in the query: count. Therefore, the result from this join should be grouped by date. Here is the complete query:

Select x1.date,
 count(distinct merchant_id) as mau
From calendar x1
Left Join
     (
       Select distinct
       Merchant_id,
       date_trunc(day,created_at) as transaction_day
        From transactions
     )x2
On x2.transaction_day between dateadd(day,-29,x1.date) and x1.date
Where x1.year in (2017,2018)
group by date;

This query returns, for each day between the years 2017 and 2018, the number of unique merchants that got involved in a transaction during the last 30 days.
